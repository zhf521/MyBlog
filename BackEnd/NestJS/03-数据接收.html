                    数据接收

                    茕茕大约 2 分钟后端Nest后端

                    在开发中，后端服务通过 HTTP 接口与前端进行数据交互。这些接口支持多种数据传输方式，每种方式都有其特定的用途和适用场景。以下是五种常见的 HTTP 数据传输方式及其在 Nest 框架中的应用。

                    我们先使用 Nest CLI 生成一个 person 模块的 crud 代码：

                    nest g resource person

                    我们将在 person 模块中演示

                    URL 参数（URL Params）

                    • 定义：URL 路径中的参数，用于指定资源。

                    • 示例：/api/user/123，其中 123 是用户 ID。

                    • Nest 接收：

                      使用 @Controller 和 @Get 装饰器，结合 :参数名 路径来获取 URL 参数。

                      // person.controller.ts

@Controller('api/person')
export class PersonController {
    @Get(':id')
    urlParam(@Param('id') id: string) {
        return `接收到的数据：id=${id}`;
    }
}

                    查询字符串（Query String）

                    • 定义：通过 URL 的 ? 后跟键值对传递数据。

                    • 示例：http://example.com/search?query=keyword&page=2。

                    • Nest 接收：

                      使用 @Query 装饰器来获取查询字符串参数。

                      // person.controller.ts

@Controller('api/person')
export class PersonController {
    @Get('find')
    query(@Query('name') name: string, @Query('age') age: number) {
        return `接收到的数据：name=${name},age=${age}`;
    }
}

                    表单数据（application/x-www-form-urlencoded）

                    • 定义：HTML 表单提交时的默认编码格式，数据以键值对形式出现在请求体中。

                    • 示例：username=jack&password=123456。

                    • Nest 接收：

                      使用 @Body 装饰器结合 encodeURIComponent 或 query-string 库来处理。

                      // person.controller.ts

@Controller('api/person')
export class PersonController {
    @Post()
    body(@Body() createPersonDto: CreatePersonDto) {
        return `接收到的数据：${JSON.stringify(createPersonDto)}`;
    }
}

                    表单数据（multipart/form-data）

                    • 定义：用于上传文件，使用分隔符（boundary）来区分不同的表单项。

                    • 示例：文件上传时，每个文件和表单字段都作为请求的一个部分。

                    • Nest 接收：

                      使用 @UseInterceptors 装饰器结合 FilesInterceptor 来处理文件上传。

                      // person.controller.ts

@Controller('api/person')
export class PersonController {
    @Post('file')
    @UseInterceptors(
        AnyFilesInterceptor({
            dest: 'uploads/',
        })
    )
    body2(
        @Body() createPersonDto: CreatePersonDto,
        @UploadedFiles() files: Array<Express.Multer.File>
    ) {
        console.log(files);
        return `接收到的数据：${JSON.stringify(createPersonDto)}`;
    }
}

                    JSON（application/json）

                    • 定义：轻量级数据交换格式，易于人阅读和机器解析。

                    • 示例：{"username": "jack", "password": "123456"}。

                    • Nest 接收：

                      直接指定 Content-Type 为 application/json，使用 @Body 装饰器来接收 JSON 数据。

                      // person.controller.ts

@Controller('api/person')
export class PersonController {
    @Post()
    body(@Body() createPersonDto: CreatePersonDto) {
        return `接收到的数据：${JSON.stringify(createPersonDto)}`;
    }
}
